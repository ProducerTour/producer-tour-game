# OPTIMAL TREE MODEL STRUCTURE FOR THREE.JS

Based on Three.js documentation and best practices:

## For InstancedMesh (Best Performance)

The ideal tree model structure for `InstancedMesh` rendering:

```
tree.glb
├── TreeMesh (single merged geometry)
│   ├── Trunk vertices (bottom)
│   ├── Branch vertices (middle)
│   └── Leaf vertices (top, using alpha cutout texture)
└── Materials
    └── TreeMaterial (single atlas texture with trunk + leaf regions)
```

### Why Single Mesh is Best:
1. **1 draw call per instance type** vs 2+ for separate trunk/leaves
2. **No positioning math needed** - geometry is pre-positioned
3. **Single material** with texture atlas = no material switches

## Recommended Model Specifications

| Property | Recommended Value |
|----------|------------------|
| Polygon count | 500-2000 triangles |
| Texture atlas | 1024x1024 or 512x512 |
| UV mapping | All UVs in single atlas |
| Origin point | Base of trunk (Y=0) |
| Scale | 1 unit = 1 meter |
| Format | GLB (binary GLTF) |

## LOD Levels for Trees

```javascript
// Three.js LOD setup
const treeLOD = new THREE.LOD();
treeLOD.addLevel(highDetailMesh, 0);      // 0-40m: 2000 tris
treeLOD.addLevel(mediumDetailMesh, 40);   // 40-80m: 500 tris
treeLOD.addLevel(billboardSprite, 80);    // 80-150m: 2 tris (cross billboard)
treeLOD.addLevel(emptyObject, 150);       // 150m+: culled
```

## Billboard Imposter (Cheapest Far LOD)

For distant trees, use cross-billboards (2 intersecting planes):
- Render tree to texture from multiple angles
- Use 2 planes at 90° angles
- Automatic rotation toward camera (Sprite)

## Texture Optimization

| Format | Use Case |
|--------|----------|
| PNG | Alpha cutout foliage (alphaTest) |
| KTX2 | GPU-compressed (best performance) |
| WebP | Good compression, wide support |

### Material Settings for Foliage:
```javascript
material.alphaTest = 0.5;      // Hard cutout (no sorting)
```

## Performance Tips from Docs

1. **Use `InstancedMesh`** for repeated objects (trees, grass, rocks)
2. **Use `LOD`** for distance-based detail reduction
3. **Use `BufferGeometryUtils.mergeGeometries()`** for static objects
4. **Set `matrixAutoUpdate = false`** for static objects
5. **Use `Layers`** for selective rendering/raycasting
6. **Pre-compile shaders** with `renderer.compile(scene, camera)`
7. **Use `frustumCulled = true`** (default) for automatic culling
8. **Minimize draw calls** by merging geometries and using atlases
